NAME

plotdat − plot graphs from many data files

SYNOPSIS

plotdat -a axis-names [-cDhvV] [-f axis-format ] [-m col-numbers ] [-M col-names ] [-o word ] [-l legend-spec ] [-s data-sep ] [-t title ] [-x -y -z range ] [--3D-mode] [--catenate] [--dirmode-gen pattern ] [--dirmode-gen-delim dir-sep ] [--filemode-gen pattern ] [--filemode-gen-delim file-sep ] [--gnuplot-options options ] [--header-parser options ] [--viewer-position position-spec ] data files ...

DESCRIPTION

plotdat takes the arguments and data files provided to create single graphs for each data file by default, or a single graph representing all data files. Further capabilities such as 3D are to be implemented. It invokes ’gnuplot’ with a set of generated arguments and variables created from the options passed to ’plotdat’. There are ’gnuplot’ command source files for each operational mode which contain a predefined sequence of commands. These files can be easily edited to adapt the output to one’s needs.

SHORT OPTIONS

-a

Axis names, first three strings delimited by ’\’ characters.

-c

Create single image from data files (catenate).

-D

Debug output.

-f axis-format

Axis number format. Uses gnuplot(1) syntax.

-h

Help output.

-m col-numbers

Specify data columns to plot from data file.

-M col-names

Specify the names of data columns specified with ’-m’.

LONG OPTIONS

--3D-mode

Run gnuplot in 3D plotting mode.

--catenate

Create single image from data files.

--dirmode-gen pattern

Select directory names using numbers delimited by spaces.

--dirmode-gen-delim dir-sep

Use dir-sep character to split directory path names. Default is ’/’.

--filemode-gen pattern

Select filename pieces using numbers delimited by spaces.

--filemode-gen-delim file-sep

Use file-sep character to split filenames into a set of strings. Default is ’_’.

--gnuplot-options options

Pass options to gnuplot invocations.

--header-parser options

Pass options to header parser invocations. See plotdat_hp(1) for a description.

--viewer-position position-spec

Viewer position specified in gnuplot(1) syntax.

EXAMPLES

Plotting graphs for each data file provided using different columns:

plotdat -a "Voltage (V)\Current (A)" -m "3" -M "I_AB" *.dat
plotdat -a "Voltage (V)\Current (A)" -m "2 3" -M "I_CD I_AB" *.dat

Plotting a single graph for each data file provided with labels:

plotdat -ca "x axis\y axis" -m "2 4" -M "sinx cosx" *.dat

Using the filemode-gen options:

plotdat -a "Voltage (V)\Current (A)" -m "2 3" -M "I_CD I_AB" *.dat --filemode-gen "2 3"

If the file names *.dat have the following format:

experiment_1A_T=50K.dat
experiment_1B_T=50K.dat
experiment_2A_T=100K.dat
experiment_2B_T=100K.dat

The curve labels in the legend will have "1A_T=50K", "1B_T=50K" and so on, in their names. With the labels being generated as "I_CD_1A_T=50K", "I_AB_1A_T=50K", "I_CD_1B_T=50K", "I_AB_1B_T=50K" and so on, allowing the curves to be identified.

Using the header parser:

plotdat -a "Voltage (V)\Current (A)" -m "2 3" -M "I_CD I_AB" *.dat --header-parser "-t,Vout"

If the headers contain entries like "Vout: 10.0 mV", the parser will by default print the number of header lines for each file and a line "Vout=10.0mV" where all spaces are stripped. See plotdat_hp(1) for details about the regular expressions and how they are overriden. Since the filemode-gen option was not passed, the curve labels will not be named using the file name, only the output from the parser, which in this case could be "Vout=1mV", "Vout=2mV", "Vout=3mV" and so on.

Using many features:

plotdat -ca "Voltage (V)\Current (A)" -m "2 3" -M "I_CD I_AB" *.dat --header-parser "-t,Vout1\Vout2" --filemode-gen "2 3"

With the information from the two previous examples, this command will create a single graph, with all curves having names "I_CD_1A_T=50K Vout1=1mV_Vout2=54nV", "I_AB_1A_T=50K Vout1=1mV_Vout2=54nV", "I_CD_1B_T=50K_Vout1=2mV Vout2=72nV" and so on.

Whereas without the -c option a graph for each data file will be created, and the names would be "I_CD_1A_T=50K Vout1=1mV Vout2=54nV", "I_AB_1A_T=50K Vout1=1mV Vout2=54nV" in one file, then "I_CD_1B_T=50K Vout1=2mV Vout2=72nV" and so on.

FILES

onedf.plt

gnuplot command code for creating single 2D graphs for each data file provided.

multidf.plt

gnuplot command code for creating one 2D graph from all data files provided.

ENVIRONMENT

PLOTDAT_PLTSRC

Used by plotdat to find the gnuplot command files for creating the graphs.

DIAGNOSTICS

The easiest way to get familiar with plotdat is to run it without any arguments and see what it requires to produce output.
